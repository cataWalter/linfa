## Implementation Order

1. **Unit Tests** (foundation, no Flutter rendering needed)
   - Models → Utils → Enums → Repositories → Providers

2. **Widget Tests** (isolated component testing)
   - Shared widgets → Feature widgets → App widget

3. **Screen Tests** (full screen integration with mocks)
   - Home → Plants → Reminders → Growth → Settings

4. **Integration Tests** (real device flows)
   - App flow → CRUD → Reminders → Growth → Export/Import → Search

5. **Golden Tests** (visual regression)
   - Key components → Themes

## Key Testing Strategies

- Use existing mock infrastructure (MockIsar, MockHive, Mockito mocks)
- Use existing test data factories for consistent test data
- Override Riverpod providers in tests with mock repositories
- Test edge cases: empty states, error states, loading states
- Test platform-specific behavior where applicable

## Running Tests

```bash
# Unit tests
flutter test test/unit/

# Widget tests
flutter test test/widget/

# Screen tests
flutter test test/screen/

# Golden tests (update golden files)
flutter test test/golden/ --update-goldens

# Integration tests (requires device)
flutter test integration_test/

# All tests
flutter test

# Coverage report
flutter test --coverage
genhtml coverage/lcov.info -o coverage/html
```
